# v2.4.0 Performance — PPR + use cache Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enable Partial Prerendering (`cacheComponents: true`) in Next.js 16 — static shells prerendered at build, dynamic content streamed at request time — reducing TTFB across all four views.

**Architecture:** PPR is blocked by 5 page/special routes that carry `export const dynamic` segment configs incompatible with `cacheComponents`. 9 API routes also have `export const runtime = "nodejs"` but are request handlers — PPR does not apply to them and they are left unchanged. The 5 page routes are migrated: 4 static ones drop their `force-static` config (Next.js 16 infers static by default for GET handlers with no dynamic deps), and the admin/telemetry page swaps `force-dynamic` for a `noStore()` call at the data-fetch level.

**Tech Stack:** Next.js 16 App Router, `cacheComponents: true`, `unstable_noStore`, TypeScript strict, pnpm, Vitest.

## Global Constraints

- Branch: `feat/v2.4.0-performance-ppr` (already checked out)
- All tests: `pnpm test` — must stay green after every task
- Build gate: `pnpm build` — PPR errors surface as build-time errors, not runtime
- Never remove `export const runtime = "nodejs"` from API routes — they need Node.js for streaming/AWS SDK
- Commit style: `perf(ppr): ...` per conventional commits
- No Co-Authored-By trailers
- Each task is one atomic commit — if a route breaks, revert that commit only

## Task 6: Migrate admin/telemetry — swap force-dynamic for noStore()

**Files:**
- Modify: `src/app/admin/telemetry/page.tsx`

This is the highest-risk change. The page needs to remain dynamic (reads live Redis data) but can't use the `force-dynamic` segment config with PPR. The correct approach is `noStore()` at the data-fetch level — this opts out of caching for that specific data function while leaving PPR free to generate a static shell.

- [ ] **Step 1: Read the full file**
```bash
cat src/app/admin/telemetry/page.tsx
```

- [ ] **Step 2: Add noStore import and call**

At the top of the file, after existing imports, add:
```ts
import { unstable_noStore as noStore } from "next/cache";
```

Find the `fetchAll` async function (it calls Redis). Add `noStore()` as the first line inside it:
```ts
async function fetchAll(since: number): Promise<TelemetryEvent[]> {
  noStore();   // ← ADD THIS — opts this data function out of caching
  const results = await Promise.all(KIND_LITERALS.map((k) => fetchKind(k, since)));
  return results.flat().sort((a, b) => a.ts - b.ts);
}
```

- [ ] **Step 3: Remove the two segment config lines**

Remove:
```ts
export const runtime = "nodejs";   // ← REMOVE
export const dynamic = "force-dynamic";  // ← REMOVE
```

Keep `runtime = "nodejs"` ONLY if it's needed for Node.js-specific APIs — check the file. If `timingSafeEqual` or other Node.js crypto is used, keep it. If the file only uses Redis (Upstash REST client, which works without Node.js runtime), remove it.

- [ ] **Step 4: Run build and verify admin page still works**
```bash
pnpm build 2>&1 | grep -E "error|admin|telemetry" | head -15
```

If the build fails on this file — revert and keep `runtime = "nodejs"` while only removing `force-dynamic`:
```bash
git checkout src/app/admin/telemetry/page.tsx
# re-apply only removing force-dynamic, keep runtime = "nodejs"
```

- [ ] **Step 5: Run tests**
```bash
pnpm test 2>&1 | tail -8
```

- [ ] **Step 6: Commit**
```bash
git add src/app/admin/telemetry/page.tsx
git commit -m "perf(ppr): swap force-dynamic for noStore() in admin/telemetry, remove segment configs"
```
